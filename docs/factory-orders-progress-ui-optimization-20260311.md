# 工厂订单状态进度条 UI 优化记录（2026-03-11）

## 1. 调研结论
- 页面位置：`/orders/factory`（组件：`src/views/FactoryOrders.tsx`）。
- 现状实现：进度节点使用圆形图标 + 文本，步骤连接使用三角形箭头。
- 主要问题：
  - 视觉层级偏弱，节点与连接关系不够直观，长链路可读性一般。
  - 进度状态（待处理/部分完成/已完成/异常）没有在“节点容器层”形成明显区分。
  - 可点击节点反馈较弱，交互暗示不够明确。

## 2. 设计目标
- 保持现有业务逻辑与交互行为不变，仅升级视觉表达。
- 强化状态识别和步骤连续性，提升扫描效率。
- 兼顾桌面与移动端，避免因节点变宽导致体验退化。

## 3. 方案与改动

### 3.0 版本标记（按当前对话）
- 当前激活 A（紧凑版）：图标上置 + 小字号 + 缩窄节点宽度（桌面固定 116，移动固定 108）。
- 备选 B（视觉增强版）：在紧凑布局基础上强化状态视觉风格（状态顶条、渐变色连接、图标立体化）。
- 备选 C（中性商务版）：保留紧凑布局与状态辨识，降低渐变与阴影强度，使其与现有后台整体风格一致。

### 3.1 结构语义增强（TSX）
- 文件：`src/views/FactoryOrders.tsx`
- 改动：
  - 为每个节点统一计算 `progressStateClass`（`pending/completed/partial/overcut`）。
  - 将该状态 class 同步到节点容器、图标、状态文案、连接线。
  - 异常节点图标由通用时钟改为 `ExclamationCircleOutlined`，降低误判。

### 3.2 视觉系统升级（CSS）
- 文件：`src/styles/factory-orders.css`
- 改动：
  - 节点从“悬空图文”改为“卡片化节点”（边框 + 渐变底 + 圆角）。
  - 状态色从仅图标扩展到节点容器与文本层：
    - `completed`：绿色体系
    - `partial`：橙色体系
    - `overcut`：红色体系
    - `pending`：中性灰体系
  - 连接线从三角箭头改为“圆角短线 + 末端圆点”，降低老旧感并提升连续性。
  - 可点击节点增加悬停抬升、阴影、边框高亮，提升可操作性提示。
  - 移动端缩窄节点和连接线宽度，保留横向滚动但减少拥挤感。

### 3.3 视觉增强版（B）补充
- 文件：`src/styles/factory-orders.css`
- 改动：
  - 为节点增加顶部状态高亮条，强化“扫一眼识别状态”的能力。
  - 节点背景改为更有层次的渐变，状态边框色更明确。
  - 图标从方角图标块调整为高对比圆形徽章，提升可识别性。
  - 连接线升级为状态渐变色并增加轻阴影，流程连续感更强。
