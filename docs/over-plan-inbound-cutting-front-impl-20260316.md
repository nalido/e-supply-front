# 裁床超用/超裁二次补录前端记录（2026-03-17）

## 1. 本轮目标
- 保留后端现有 `usage*` / `overCut*` 契约不变。
- 主完工弹窗默认不展示超用原因、超用说明、超裁原因、超裁说明。
- 用户点击“确定”时，前端先根据当前输入检测是否超用或超裁。
- 只有命中超用或超裁时，才弹出第二个确认弹窗，强制补录对应原因与说明。

## 2. 交互调整
### 2.1 完工主弹窗
- 仅保留数量录入与差异提示。
- Alert 继续实时提示计划用量、实际用量、计划件数、本次实裁件数。
- 实时提示里明确说明：
  - 超用时需要补录超用原因和说明
  - 超裁时需要补录超裁原因和说明
  - 超裁不会自动扩张成品待入库

### 2.2 点击“确定”后的分支
- 未超用且未超裁：
  - 直接提交完工接口
- 超用或超裁：
  - 弹出“补录超用 / 超裁原因”二次弹窗
  - 只展示当前命中的字段
  - 超用时必填：`usageReasonCode` + 1 个说明输入
  - 超裁时必填：`overCutReasonCode` + 1 个说明输入

### 2.3 详情展示
- 待裁详情、已裁详情继续展示：
  - 超用原因
  - 超用备注
  - 超裁原因
  - 超裁备注
