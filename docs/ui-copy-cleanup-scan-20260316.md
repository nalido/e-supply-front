# 新版 UI 页面文案扫描与清理记录（2026-03-16）

## 1. 调研结论

### 1.1 调研范围
- 仓库：`e-supply-front`
- 范围：`src/views`、`src/components`、`src/menu.config.tsx`、`src/router.tsx`
- 扫描关键词：`本轮`、`联调`、`收口`、`当前仅`、`演示环境`、`操作指南`、`路径提示`、`内部说明`

### 1.2 扫描结果摘要
本次扫描发现新版 UI 中仍有几类不适合正式产品页面展示的文案：
1. **联调/收口类说明**：直接描述“当前可联调链路”“本轮仅收口”等研发过程信息。
2. **页面结构说明**：如“统一按 PageHeader → FilterBar → TableToolbar 组织”等，属于设计/开发实现说明，不是业务用户关心的信息。
3. **演示环境提示**：如“当前为演示环境，删除操作未开放”，表达方式偏内部测试环境。
4. **文档型页面入口**：`/guide` 相关页面属于操作手册/图文指南，更适合留在 docs，而不是正式产品导航中。

### 1.3 本次处理原则
- 保留业务含义，移除研发过程语气。
- 页面副标题改为面向业务用户的简洁说明。
- 能力限制类提示保留，但改为正式产品语气。
- 将明显属于操作文档的 UI 入口从正式导航中移除。

## 2. 实施计划
1. 先扫描并确认所有疑似内部文案位置。
2. 逐页替换为正式、简洁、面向业务用户的产品文案。
3. 移除 `/guide` 在正式 UI 中的导航入口与路由暴露。
4. 执行本地构建验证，确认改动不影响产物生成。

## 3. 改动说明

### 3.1 已清理/替换的页面与文案

#### 1）`src/views/CuttingPending.tsx`
- 原文案：
  - `按"待裁 → 开裁 → 床次录入 → 裁剪完成"的闭环顺序推进当前可联调任务。`
  - `本轮仅收口已能联调的待裁页面，不继续深入成品车缝 → 入库链路。`
- 调整后：
  - `集中处理待裁任务，依次完成开裁、床次录入与裁剪结果登记。`
  - `支持按任务进度查看裁剪状态与用料执行情况。`

#### 2）`src/views/SampleList.tsx`
- 原文案：
  - `把列表筛选、状态推进和下大货入口收口到同一工作台，方便直接验收当前可联调链路。`
- 调整后：
  - `统一查看样板进度、处理状态流转，并快速发起下大货。`

#### 3）`src/views/SampleDetail.tsx`
- 原文案：
  - `本页已将基础信息、数量结构、BOM/尺寸/附件与成本拆解收口到一页，便于业务在真实数据下连续检查。`
- 调整后：
  - `在同一页面查看样板基础信息、数量结构、BOM、尺寸、附件与成本明细，便于快速核对。`

#### 4）`src/views/FinishedGoodsPendingReceipt.tsx`
- 原文案：
  - `统一按 PageHeader → FilterBar → TableToolbar → 表格组织，状态筛选与分组全部收口到筛选条。`
- 调整后：
  - `支持按订单、款式、供应商和收货状态快速筛选待收货记录。`

#### 5）`src/views/StockingPurchaseInbound.tsx`
- 原文案：
  - `将状态筛选收口到 FilterBar，列表区使用统一 TableToolbar 和批量动作条。`
- 调整后：
  - `按物料类型、入库状态和关键字快速查询采购入库记录。`

#### 6）`src/views/MaterialIssueDetails.tsx`
- 原文案：
  - `后续将支持编辑：${record.materialName}`
  - `当前为演示环境，删除操作未开放`
- 调整后：
  - `暂不支持编辑：${record.materialName}`
  - `暂不支持删除已生成的出库明细`

### 3.2 从正式 UI 中移除的文档型入口

#### 7）`src/menu.config.tsx`
- 移除菜单项：`/guide -> 操作指南`

#### 8）`src/router.tsx`
- 移除路由：
  - `/guide`
  - `/guide/:sectionId`

说明：`OperationGuide` 与 `guide/GuideSectionPage` 更适合作为文档内容保留在 docs 体系，而不是正式业务 UI 的一部分。本次先移除正式导航与路由暴露，避免业务用户进入文档型页面。

1. 仓库中仍保留 `OperationGuide.tsx`、`guide/GuideSectionPage.tsx` 和 `guide-data.ts` 文件，但已经不再通过正式菜单和路由暴露。
